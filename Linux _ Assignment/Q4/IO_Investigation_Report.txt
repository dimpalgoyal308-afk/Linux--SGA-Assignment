2. Technical Observations and how linux manage file I/O
   File Descriptors:
   In Linux, whenever a program opens a file, a stream, or a network link, the operating system assigns it a tracking number called a File Descriptor (FD). 

- The Three Standard Streams:

1. File Descriptor 0 (Standard Input ): Receives typed input from your keyboard.
2. File Descriptor 1 (Standard Output ): Sends normal command text results to your screen.
3. File Descriptor 2 (Standard Error): Sends error problem messages to your screen.

- How Redirection Solves Logging Issues:
The character '>' tells Linux to intercept the normal text output (FD 1) and send it into a text file instead of printing it to the black screen. 
Adding a '2>' specifically tells Linux to trap error messages (FD 2) and pipe them into a dedicated error log file. This keeps normal data and error messages separated, making it easy to troubleshoot broken applications.
